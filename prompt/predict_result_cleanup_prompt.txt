================================================

1. You only edit `transcription`.
2. Do not output any other field.

ALLOWED REPAIRS
================================================

You may repair only clear format noise inside `transcription`, for example:

- stray Markdown code fences such as ```json ... ```
- wrapper phrases accidentally copied into the field, such as
  "Here is the cleaned result" or "Below is the repaired JSON"

- obviously broken `[[MATH:...]]` wrappers when the intended wrapper is already clear
- doubled escaping around LaTeX commands when a conservative cleanup is obvious

Use `raw_output` only as same-record evidence when the current `transcription`
is visibly polluted by formatting artifacts.

MATH HANDLING
================================================

Be conservative with mathematical content.

- Preserve genuine handwritten math content.
- Preserve existing `[[MATH:...]]` fragments when they are already valid.
- If a math wrapper is clearly malformed but the intended math span is obvious, repair it.
- Keep narrative text outside `[[MATH:...]]` unless the math span is already clear.
- Do not re-typeset content for style alone.


================================================
JSON OUTPUT RULES
================================================

The final response must be raw JSON text.

1. Top-level value is exactly one JSON object.

- Markdown code fences
- explanations
- notes
- comments

- preserve as much original content as possible
- make only the smallest safe correction
- avoid speculative rewriting

If no correction can be made safely, return the original `transcription`.
